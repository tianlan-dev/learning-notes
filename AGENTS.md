
这份笔记给 AI 助手快速理解当前 Obsidian vault 的结构和使用约定。

## Vault 概况

- `README.md` 是全局总览入口，包含了这个Vault的用处说明。
- 笔记多使用中文标题，目录按学科和主题分层组织。

## 读取建议

1. 优先查看 `README.md`。
2. 需要理解某个领域时，先读取对应分类下的 `总览`、`概括`、`简介` 类笔记。
3. 修改笔记前先搜索相关主题，避免重复创建相近内容。
4. 新增笔记时尽量放入已有分类目录，而不是直接放在根目录。

## 写作约定

- 使用清晰的 Markdown 标题层级。
- 同一主题尽量保持一篇主笔记，必要时再拆分子主题。
- 保留中文语境和已有命名风格。
- 对历史、科学等内容，优先写结构化摘要，再补充细节。
- 所有内部笔记链接都使用 GitHub 支持的标准 Markdown 相对链接，格式为“方括号显示文本 + 圆括号相对 md 路径”；不要使用 Obsidian 专用的双中括号 wiki 链接。
- 链接目标包含中文、空格或特殊字符时，使用 URL 编码后的相对路径，确保在 GitHub 上可以点击。

## Codex / AI 操作约定

- 当 Codex 或同类 AI 助手操作这个 vault / repo 时，永远优先使用 Obsidian MCP。
- 读取、搜索、创建、移动、重命名、修改笔记时，优先调用 Obsidian MCP 工具，而不是直接用 shell 文件操作。
- 只有当 Obsidian MCP 无法完成任务，或任务明显不属于 Obsidian 笔记操作时，才考虑使用其他工具。
