# 历史文明与国家笔记结构规范

## 核心原则

- 物理目录以宏观地区为主，README 同时提供“按地区”和“按文明 / 历史共同体”导航。
- 文明区、历史区域、帝国、族群共同体、殖民地和现代国家是不同对象，必须在入口和正文中标明类型。
- 跨越现代国界的文明、帝国和共同历史过程只维护一篇规范主笔记；国家页提供本地视角并回链，不复制完整正文。
- 政权更替只是文明史的一条轴线，必须同时观察语言文字、宗教思想、制度法律、人口族群、经济网络和空间中心。
- 现代国家可以承接古代地域和文化资源，但不得被写成全部古代文明的唯一、无争议直系继承者。

## 区域 README 固定骨架

1. `## 范围与概括`
2. 全史演进图
3. `## 文明与历史空间入口`
4. `## 现代国家与政治实体入口`
5. `## 区域共同史与跨境专题`
6. 重要转折或关键时间节点
7. 关键辨析
8. `## 上级与相邻区域`

区域 README 只保留范围、主线、关键差异和入口；细节事实放入下级笔记。

## 文明 / 历史共同体 README 固定骨架

1. `## 范围与对象`：对象类型、核心地理、古今同名关系、与现代国家的关系。
2. `## 一句话全史`：概括起源、最大断裂和现代承接。
3. `## 政治演进图`：只画政权、控制和政治共同体关系。
4. `## 文明连续性`：至少覆盖语言文字、宗教思想、制度法律、人口族群、经济网络、空间中心。
5. `## 阶段导航`：时间、对象性质、核心范围、承接、变化、并行 / 后继。
6. `## 关键断裂与现代承接`。
7. `## 跨区域转折`。
8. `## 上级与相关文明`。

## 现代国家 README 固定骨架

- 前史部分明确写成“本国现辖地域视角”或“共享前史”，不得宣称独占跨境文明。
- 从现代国家形成开始，重点维护边界、殖民 / 帝国治理、独立、宪制、政体、社会和外交。
- 分治、解体和并立必须画成分叉，不得只链接其中一个后继国家。
- 若古代文明已有区域规范主笔记，国家页只保留摘要、当地遗址 / 区域过程和规范入口。

## 阶段页固定骨架

- `## 时间`
- `## 范围 / 政治中心`
- `## 对象性质`
- `## 概括`
- `## 统治结构 / 主要主体`
- `## 重要事件`
- `## 延续了什么`
- `## 改变或断裂了什么`
- `## 前接、并立与后接`
- `## 跨区域链接`
- `## 上级`

不得用“本节点属于某主线”“后续继续扩充”等占位句代替具体关系。

## 内容完整度与分级详写

- 核心王朝、帝国或文明节点应以约 1500—3000 字的正文说明完整主线；普通历史阶段以约 800—1500 字为参考。篇幅服从内容，不用重复文字凑数。
- 区域与文明 README 继续承担范围、主线和导航，不复制下级笔记的事件、制度与世系细节。
- 已有内容不得因统一模板而删减；补全时应保留原事实、例子、图片、资源和有效内部链接。

## 王朝、帝国与政权页内容契约

单一政权页应根据实际情况覆盖：

1. 建立背景与崛起机制。
2. 按阶段展开的发展过程。
3. 统治结构、财政军事基础与主要政治主体。
4. 短阶段至少三个、长时段至少五个实质事件，写明前因、过程、结果和影响。
5. 鼎盛或长期维持统治的条件。
6. 衰落的结构因素、外部压力与直接触发因素。
7. 灭亡、解体、禅让、征服或制度转型的具体过程。
8. 前接、并立、分裂与后继关系。

兴衰分析不得只列单一原因；应尽量同时检查政治制度、继承合法性、财政经济、军事外交、人口社会与环境条件。

## 君主与统治者完整度

- 单一君主制政体应列完整的公认统治者，严格按在位时间排序；共治、复位、废立和有争议者在备注中说明。
- 不使用“后期诸帝”“后期诸王”等合并项代替可考统治者。资料不明时写“约”“不详”或“存在争议”，不得猜测。
- 短世系直接放在主笔记；长世系建立独立专页，主笔记保留阶段摘要并双向链接。
- 中国政权世系尽量包含顺序、姓名、庙号、谥号、年号、在位、生卒、与前任关系和关键事件。
- 外国君主制世系至少包含顺序、统治者、称号或王室、在位时间、与前任关系、关键事件和备注。
- 共和国、殖民地、占领期分别列国家元首、政府首脑、殖民行政首脑和实际权力结构，不套用君主表。
- 多政权综合页分别说明各政治体及关键统治者；不得为了表格整齐制造一条不存在的统一世系。

## 事件、制度与专题页内容契约

- 事件页应写明背景、参与者、阶段过程、关键转折、结果和长期影响；争议数字与责任归属应明确说明不确定性。
- 制度页应说明形成条件、运作机制、参与主体、历代变化、实际效果和例外。
- 族群、语言、宗教与文化页应说明形成与传播、区域差异、互动融合和争议，不把文化影响写成确定血缘或政治继承。
- 世界与区域通史负责比较“条件—触发—扩散—结果—长期影响”，不重复维护各地区的完整世系。

## 内容验收

- 古代年代、共治、僭位、口述传统和争议谱系至少交叉核对两份权威学术或机构资料。
- 现代“至今”页面标明事实核验截止时间；现任人物优先核对政府、议会、王室或国际组织官方名录。
- 正文不建立外部参考资料章节或外部链接清单。
- 每批修改后检查世系连续性、时间重叠、前后节点、占位句、重复正文、frontmatter、根相对 URL 编码链接、孤立页和 Mermaid 语法。

## Mermaid 关系语义

| 关系 | 推荐写法 |
|---|---|
| 普通继承或发展 | `A -->\|继承 / 发展\| B` |
| 征服或灭亡 | `被征服者 -->\|年份 征服者灭其政权\| 征服者` |
| 分裂 | 来源节点分别指向多个后继节点 |
| 并立或竞争 | 分成并行节点，用带标签箭头说明竞争、联盟或控制范围 |
| 占领与宗主 | 政体线与实际控制线分开，边标签写明“占领”“保护”“宗主” |
| 文化影响 | 标签写“影响”“传播”“可能影响”，不得冒充政治继承 |
| 谱系有争议 | 正文和边标签明确写“关系有争议” |

时间重叠的王朝、占领、殖民统治、共和国和社会文化专题不得画成无标签单线先后。

## 导航与属性

- 新增或修改笔记至少保留 `status: 已整理` 和 `title`。
- 总览可增加 `type`，取值可用“区域史”“文明史”“国家史”“政权史”“共同史与跨境专题”“维护记录”。
- 每个 README 必须链接直接父级 README；阶段页应链接所属总览。
- 内部链接使用根相对、URL 编码的标准 Markdown 链接。
